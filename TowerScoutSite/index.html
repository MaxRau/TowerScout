About

Our Mission

Legionella bacteria can cause severe respiratory disease with a 1 in 10 fatality rate. These bacteria grow in the water of cooling towers of large buildings, especially when these systems are off. Turning these systems back on can blow bacteria mist around the neighborhood, causing disease outbreaks. The current Legionella outbreak investigation processes include:

  • Investigators manually search satellite imagery or a pre-existing registry for nearby cooling towers.
  • investigators test towers for bacterial contamination and treat them to stop the source of the outbreak

Manually searching satellite imagery is hard. This is why cooling tower registries are widely considered one of the best practices in preventing and improving the response to legionellosis outbreaks. But they are labor-intensive and expensive to create, which is why very few U.S. cities have them. Public health is already overburdened with the COVID response and doesn’t have bandwidth to handle a big surge in outbreaks. Our goal is to create a tool that automatically detects cooling towers from satellite imagery.

Training Data

Our source data comes from cooling tower registries for a few cities. We pull satellite images based on cooling tower locations, and annotate images with horizontal bounding boxes.

  • New York
  • Philadelphia
  • Seattle, Atlanta, Houston, Chicago

Data Augmentation

We leverage Roboflow's built-in functions for basic data augmentation techniques, including flip, rotation, blur and noise. To generate synthetic data, we randomly copy and paste cooling tower objects onto backgrounds without cooling towers. Previous research has shown that copy-paste is an efficient way to create large annotated training data. We tested background images from different cities and cooling tower objects from different cities. We also applied blending effects to smooth out the edges of pasted objects. The results showed that blending improved performances.

YOLOV5

Model comparison

Towerscout is built on an object detection algorithm called YOLOv5. (“You Only Look Once.”) As the title suggests, the model is a “single shot detector”, meaning it looks at the whole image at once and tries to identify the relevant areas of interest. This makes it extremely fast, while still providing fantastic performance.

We specifically leveraged the largest version of YOLOv5, called YOLOv5 XL, with pretrained weights from the COCO dataset. The model was fine-tuned for 100 epochs on a V100 GPU, and performance was evaluated using a custom evaluation metric which is aligned to the true evaluation process of our stakeholders. An optimal inference confidence threshold range was also established and used to create the default search band within the user inference.

Performance

Our objective was to minimize the False Negatives, as these would be the most detrimental to our stakeholders.

Architecture

Towerscout is hosted on AWS in Python flask, and uses either Google Maps or Microsoft Bing maps as a UI and backend tile source. Place outline data is retrieved from OpenStreetMap, and zip code searches are enabled through 2010 census zip code.

User Guide

Frequently Asked Questions

  • How to search a new area?

    In the top bar, we can search for a place, and then zoom and pan as with most maps. We can also center a particular point and ask to mark a boundary of perhaps 200 meters. Lastly, we can define a search area by drawing an arbitrary polygon on the screen.

  • How to save search results?

    We ask the server to execute our search. It will download satellite image tiles of the area we marked, and feed them to our detection algorithm. When the results come back, we can view them individually on the map and click them to show their address and confidence level. We can unselect towers that we think are false positives, and we can mark additional objects that we think the tool missed. The review bar on the right can help us quickly flip through tiles or detections. When we are satisfied with the results of our search, we can download the results and receive them as both a CSV file … and a KML file to view in Google MyMaps or Google Earth.

  • How to label new training data?

    The tool has a labeling mode. When we switch into this mode, the tool will show all results on any tile scanned, regardless of whether they are in the specified boundary or not. It will also lower the confidence threshold to make sure we are not missing any detections it made. We can now flip through all tiles to mark any false negative – missing towers, that is – and after that process is complete we can flip through all detections to root out any false positives. When we are done, we download the dataset and we can see that we have our data neatly organized in labelled images for further training, and empty tiles and coded results for further work.
